# DS 입상덕트 작업현황 웹앱

엑셀 입면도(`DS 설치현황` 시트)를 **모바일에서 직접 색을 수정**하는 웹앱으로 옮긴 것.
현장 관리자·팀장이 셀을 눌러 작업상태를 실시간으로 갱신하고, 필요 시 엑셀로 내보낸다.

## 구성 파일
| 파일 | 설명 |
|---|---|
| `index.html` | 앱 화면 (이 파일을 열거나 호스팅) |
| `app.js` | 앱 로직 (격자·편집·줌·메모·카운팅·내보내기) |
| `styles.css` | 스타일 |
| `seed.js` / `seed.json` | 엑셀에서 추출한 기준 데이터(파트 100·셀 2900) |
| `config.js` | Supabase 접속 설정 (비우면 로컬 모드) |
| `import_xlsx.py` | 원본 엑셀 → seed 재생성 스크립트 |
| `supabase/schema.sql` | 클라우드 DB 테이블 생성 SQL |

## 동작 모드
- **로컬 모드(기본)**: `config.js` 가 비어 있으면 브라우저 localStorage 에 저장. 같은 PC/같은 브라우저의 여러 탭은 즉시 동기화되지만, **다른 사람과는 공유되지 않음** → 테스트용.
- **클라우드 실시간 모드**: `config.js` 채우면 Supabase 로 여러 기기·여러 사람이 **실시간 공유**. 누가 언제 바꿨는지 `change_log` 에 기록.

## 클라우드(실시간 공유) 설정 — 5분
1. https://supabase.com 가입 → **New project** 생성(무료).
2. 좌측 **SQL Editor** → `supabase/schema.sql` 내용 전체 붙여넣고 **RUN**.
3. **Project Settings → API** 에서 `Project URL` 과 `anon public` 키 복사.
4. `config.js` 의 `SUPABASE_URL`, `SUPABASE_ANON_KEY` 에 붙여넣고 저장.
5. 끝. 앱 상단 표시가 **로컬 → 실시간** 으로 바뀐다.

> RLS 정책은 "링크만 알면 누구나 수정"으로 설정됨(현장 공유 편의). 더 엄격히 하려면 schema.sql 의 policy 를 조정.

## 배포(현장 공유) — 둘 중 하나
- **가장 쉬움**: https://app.netlify.com/drop 에 `webapp` 폴더를 드래그 → 즉시 공개 URL 발급 → QR/링크로 공유.
- **GitHub Pages / 사내 웹서버**: `webapp` 폴더 정적 호스팅.
- (로컬만 볼 땐 `index.html` 더블클릭으로도 열림 — 단, 공유는 안 됨)

## 사용법
- 상단 **구역 탭**(색 구분): 북DS(FA,SA)·동DS(FA)·북DS(배기)·동DS(배기)·전체
- **검색창**: 성상·SIZE·덕트NO 입력 → 연관 라인만 표시 (예: `1200A`, `ACID`)
- **셀 탭** → 범례에서 상태 선택(+물량) → 색 변경·저장 (레이어별 상태만 노출: 입상/횡주=설치, 바닥=타공)
- **＋/－·핀치·Ctrl+휠** 줌 — **포인터(손가락) 위치 중심**으로 확대
- 층 라벨 **📝**=층 메모, 파트 헤더 **📝**=라인 메모 (내용 있을 때만 빨간 점)
- **상부/하부접점** 행: 셀 탭 → 접점 메모 입력
- **타공현황** 행: 각 라인의 바닥(타공) 완료 `완료/전체`
- 진행 요약 바: **덕트설치 %** / **바닥타공 %** 분리 집계 + 상태별 칸수
- **엑셀 ⬇**: 현재 현황을 색·표시물량 포함 xlsx 로 내려받기
- 최초 접속 시 **이름/팀** 입력(변경 이력 추적용, 한 번만)

## 데이터 정합성(v2에서 바로잡음)
- 엑셀 표시서식(`0`/`General`)을 반영해 **물량을 엑셀과 동일하게 표시**(예: 46.04→46, 14.9는 14.9)
- **대각선 = 횡주간 없음**(횡주 레이어 전용, 값 없음) → 옅게 표시
- **열수**(병합셀)·**SIZE**·성상 전 라인 반영, **상/하부접점** 행 복원
- 범례 9종을 레이어 문맥으로 구분(금일설치/금일타공, 설치완료/기설치타공 등)

## 원본이 바뀌면
새 엑셀을 받으면 `import_xlsx.py` 상단 `SRC` 경로를 바꾸고 실행:
```
python import_xlsx.py
```
→ `seed.js`/`seed.json` 갱신. (클라우드의 수정분은 그대로 유지되고, 기준 데이터만 교체됨)

## 색 체계 (범례 — 원본 이미지 직접 확인)
원본 범례는 **색 스와치가 라벨 왼쪽**에 배치되어 있습니다.

| 상태 | 색 | 적용 레이어 |
|---|---|---|
| 미설치 | 회색 `#BFBFBF` | 전체 |
| 기타 간섭구간 | 빨강 `#FF0000` | 전체 |
| 비계 간섭구간 | 핑크 `#FF8F8F` | 전체 |
| 금일타공 | 노랑 `#FFFF00` | 바닥 |
| 타공완료 | 시안 `#66FFFF` | 바닥 |
| 금일설치 | 진파랑 `#0070C0` | 입상·횡주 |
| 설치완료 | 시안 `#66FFFF` | 입상·횡주 |
| 기설치덕트 / 기설치타공 | 하늘 `#00B0F0` | 덕트 / 바닥 |
| 작업없음 | 흰색 | 전체 |
| 횡주간 없음 | 대각선 | 횡주 |

- **설치완료(시안)와 기설치(하늘)는 원본부터 서로 다른 색**입니다(별도 색 부여 불필요).
- '완료' 집계: 덕트=설치완료+기설치덕트+금일설치 / 바닥=타공완료+기설치타공+금일타공. 미설치·간섭은 미완료.
